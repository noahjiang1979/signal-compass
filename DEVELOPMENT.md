# Hermes Signal Compass — 开发者手册

## 一、项目定性

双模式 A 股 / 美股指数的**免费数据信号监控系统**，面向做空/对冲决策辅助。

- 不是量化回测平台，不生成交易信号
- 不做自动化（双击 .bat 触发）
- 输出目标：终端，表格 = CJK 等宽硬对齐
- 语言：中文

## 二、技术栈

| 层 | 选型 | 版本 | 说明 |
|----|------|------|------|
| 运行时 | Python | 3.12+ | 标准库 + pip 包，无虚拟环境要求 |
| HTTP | requests | 最新 | 唯一网络库 |
| 金融数据 | akshare | 最新 | 主要数据源（东财接口） |
| 搜索引擎 | ddgs | 最新 | DuckDuckGo CLI，用于叙事信号 |
| 表格对齐 | 自研 | cjk_table.py | CJK 感知对齐（wcwidth），中文=2 ASCII=1 |
| 启动器 | Windows .bat | — | 桌面双击，不用 cron |

**依赖安装**（用户机器已装，新人需要）：

```bash
pip install requests akshare ddgs
```

## 五、信号系统详解

### 5.1 US 模式

**三层判定：**

1. **数据层**：SPY 价格、RSI、20MA/50MA/200MA、成交量比、Mag7 分化度、板块轮动差、VIX、收益率利差
2. **加权评分**（满分 15 分）：
   - SPY 收跌 +1 | 跌破 20MA +1.5 | 跌破 50MA +2
   - 成交量 >1.2x +1 | RSI>70 +1 | RSI>80 +1.5
   - Mag7 跌幅 >5 家 +2 | 标准差 >2% +1
   - SOX 跑输 QQQ>2% +1.5 | XLE 跑赢 >2% +1
   - IWM 跑输 >0.5% +1 | VIX>25 +1
   - 利差 <0.5% +1 | Burry 叙事 +0.5
3. **判定**：<5 🟢 乐观 / 5-10 🟡 注意 / >=10 🔴 警戒 → 15 分时确认做空

**升级/降级**：满足 ≥3 项升级条件（跌破 20MA / 跌破 50MA / 放量 20%+ / Mag7 普跌 ≥5 / VIX>30）转为确认做空窗口

### 5.2 A 股模式

**五层判定：**

1. **数据层**：6 指数矩阵、涨跌家数比、成交量比、板块轮动差（银行/半导体/上证50/医药 vs 科技）、主力资金流向、北向资金（单日+3日累计）、10Y 国债、涨停分布
2. **加权评分**（满分 12 分）：
   - 普跌: 每 1 指数跌 +1（上限 6）
   - 涨跌家数比 >3:1 +1.5 | >5:1 +2.5
   - 主力净流出 >150 亿 +2 | >100 亿 +1
   - 北向净流出 >100 亿 +1.5 | >50 亿 +1；3 日累计 <-150 亿 +2 | <-100 亿 +1.5
   - 10Y 国债 >2.0% +1
3. **判定**：<=3 🟢 中性 / 4-6 🟡 注意 / 7-9 🔴 警戒 / 10-11 🛑 高风险 / 12 🚨 确认调整

**升级/降级**：创业板跌破 20MA / 涨跌比 >5:1 / 缩量 <0.8x / 北向 3 日累计流出 >150 亿 / 主力单日 >200 亿 / 10Y>2.0% — ≥3 项升为高风险

### 5.3 异常提示系统

机器只做异常检测，不写叙事。模板固定为：

```
→ 分析:
  ✅ 确认信号 / ⚠️ 观察信号
💡 今日观察: [动态选一条]
💡 我的笔记: _________________________
```

异常检测规则写死在每个模式的 if 链中，修改需要改代码。

## 六、关键设计决策

1. **20MA 而非 MACD**：更直观，A 股短线资金实际参考线，滞后性更低
2. **Sina K 线而非 Yahoo 成交量**：Yahoo A 股成交量跳动 19x，Sina 稳定
3. **北向资金 = 沪股通 + 深股通成交净买额之和**：AkShare 接口单列汇总
4. **10Y-2Y 收益率差而非 10Y-3M**：2YY=F 数据稳定，^IRX 不稳定
5. **板块轮动 = 涨跌幅差而非价格比**：差值更直观反映资金流向变化
6. **不做自动化**：用户双击 bat，系统不跑任何后台/定时任务
7. **不输出确定性叙事**：机器只标记异常用问句引导，结论由人类写
8. **表格=标签，提示=详情**：事件类信息在表格中只显示关键词短标签（≤6字），完整描述下沉到提示区。避免截断和冗余，确保一眼扫过全是干货。

## 七、数据源可靠性评级

```
🟢 极高 — Sina 实时行情 (hq.sinajs.cn)
    连续运行 6 个月零故障，反爬极松，延迟 <1s
    故障模式：返回空字符串 ""，需 try-except

🟡 中等 — Yahoo Finance、AkShare
    Yahoo：VPN 断开时 404，梯子稳定时才可用
    AkShare：依赖东财后端，改版偶发失效（概率 <5%）

🟠 不稳定 — 东财 push2 API
    push2.eastmoney.com 的 K 线接口 (Sina K 线已替代)
    涨跌家数接口偶有 format 变化（list ↔ dict 自动适配）
```

## 八、扩展指南

### 8.1 新增一个信号指标

```
1. 在数据获取阶段（run_us 或 run_a 开头）调对应接口
2. 在加权评分块中加 if 判断和得分
3. 在升级/降级条件块中加条件打印
4. 如需异常检测，在 → 分析块中加 if 链
```

### 8.2 新增一个数据源

```
1. 在文件顶部写一个新函数（如 sina_k、yahoo_c 模式）
2. 返回值保持统一格式：成功返回数值/字典，失败返回 None
3. 不要在数据获取函数内部做判定逻辑
4. 加 try-except，失败时打印到 stderr 或静默返回 None
```

### 8.3 新增一个模式

```
1. 写一个新函数 run_xxx()
2. 在 __main__ 入口加 elif 分发
3. 参考 run_us 和 run_a 的结构：
   数据获取 → 信号映射 → 加权评分 → 判定 → 异常分析 → 今日观察 → 升级/降级条件
```

### 8.4 修改判定阈值

阈值全部写在对应 if 语句中，没有集中配置。搜索 `+1`、`+1.5`、`+2` 等关键词可找到所有评分规则。

## 九、常见问题

**Q: 北向资金显示"[盘中数据延迟]"是什么情况？**
A: AkShare 接口盘中延迟 10-15 分钟，13:00 后曲线恢复正常。收盘后 15:30 数据最准确。

**Q: 10Y 国债为什么这么慢？**
A: 首次运行时 akshare `bond_zh_us_rate` 需要下载整个历史表（约 30 秒）。之后日级缓存命中，秒出。

**Q: US 模式的成交量比用的什么基线？**
A: 20 日均量（来自 Yahoo 1 年 K 线数据）。

**Q: 为什么没有备选数据源？**
A: 经过全面验证，A 股的北向资金/10Y 国债实时接口（Sina、腾讯、东财 push2）均无法稳定获取。保持 AkShare 唯一源 + try-except 兜底，比加无效 fallback 代码更优。

**Q: 怎么加自己的指标？**
A: 见第八章扩展指南。核心原则：数据获取和判责分离。

## 十、复盘功能（Review Mode）

### 10.1 设计理念

复盘是 compass.py 在**时间维度上的自然延伸**。现有框架解决了"今天的数据怎么看"，复盘解决"今天的数据和昨天比有什么变化"。

核心分工：
- 机器做：自动存档 → 趋势展示 → 异常窗口标记
- 人做：补充笔记 [涨]/[跌]/[平] 标签 → 判断信号有效性

### 10.2 三层架构

| 层级 | 触发方式 | 功能 |
|------|---------|------|
| 第一层：自动存档 | 每次运行自动触发 | 主流程末尾调用 `save_daily_review()`，将当日关键信号存入 `review/YYYYMM.json` |
| 第二层：复盘报告 | `python compass.py review` 或选中 compass.bat 菜单 3 | 读取最近 30 天数据，输出得分趋势线 + 异常窗口 + 笔记回顾 |
| 第三层：信号闭环 | 手动在笔记中标注 | 笔记里写 `[涨]` / `[跌]` / `[平]`，`calc_accuracy()` 自动统计命中率 |

### 10.3 自动存档规则

- 存档位置：`review/YYYYMM.json`（按月分文件）
- 单日去重：同一天多次运行只保留最后一次
- 存储内容：模式、得分、满分、风险等级、关键指标快照、分析摘要、观察、笔记
- 每次约 1KB，一年不到 400KB

### 10.4 笔记联动

- 每次运行主模式时，顶部自动显示**前一天**的笔记（如果存在）
- review 模式中按时间线串起所有笔记
- 标记方式：在笔记中写 `[涨]` 或 `[跌]` 或 `[平]`，供未来命中率统计使用

### 10.5 技术实现要点

```python
# 存档（运行即触发，无感）
save_daily_review({"mode":"us","score":score,"max":max_score,...})

# 显示昨天笔记
yn=get_yesterday_note()
if yn: print("  🖋 昨日笔记：{}".format(yn))

# 底部提示
print("  🧑 双击 compass.bat 或选菜单 3 看历史趋势")
```

### 10.6 注意事项

- 第一次运行 review 模式时，如果数据不足 2 天会提示"还没有复盘数据"
- 趋势线的 `fixed_max` 参数固定为当前模式的满分（US=15, A=12），确保不同天数的趋势线可比
- 异常窗口的阈值（默认连续 3 天升降）可调整 `detect_windows(scores, threshold=N)` 参数
| - 命中率计算依赖人工标注，标注前返回 None

## 十一、Portfolio 模式（自选组合池）

### 11.1 功能概述

`python compass.py portfolio` 对自选池每只股票独立计算组合信号（MACD+BOLL+KDJ），结合事件检测和异常偏离检测，输出相对大盘强弱矩阵。

### 11.2 数据流

```
portfolio.json → 代码列表
sina_q() → 实时行情 → 相对大盘（基准: 上证+科创50+SPY）
sina_k(code, 35) → 35天日K线 → get_signal() → 组合信号
check_events() → 东财直连事件 → 短标签（表格）+ 长描述（提示区）
count_anomaly_ann() → 异动公告次数（近3月）
```

### 11.3 表格列定义

| 列 | 内容 | 来源 |
|----|------|------|
| 代码 | 股票简称（前4字） | Sina 实时行情 name |
| 价格 | 当前价 | sina_q() price |
| 涨跌 | 当日涨跌幅 | sina_q() chg |
| 相对大盘 | 个股涨跌 - 上证涨跌 | 计算 |
| 观察 | 🟢🟡🔴 + 强弱标签 | rel 阈值（>1→⭐抗跌, >0.5→抗跌, >-0.5→跟跌, else→领跌） |
| 信号 | 组合信号 emoji + 文字 | get_signal() 输出 |
| 事件 | 标签化事件（🔒质押等）或 🟡异常偏离 | check_events() tag_str / anomaly_tag |

### 11.4 事件系统

详见十二章。核心原则：**表格=标签，提示=详情**。

### 11.5 颜色覆盖

当「信号」列触发 🔴严重异动/预警/回避 时，观察栏颜色强制降级：
- `col`：🟢 → 🟡
- `hint`：去掉 ⭐ 前缀（"⭐抗跌" → "抗跌"）

### 11.6 异常偏离检测

- 交易所标准：连续3日累计涨幅≥20%
- 表格显示：🟡异常偏离
- 提示区显示：交易所要求发异动公告 | 近3月触X次（不重复标签）

### 11.7 提示区排序优先级

1. 🔴严重信号（含质押/减持双重风险）→ 附注：异动公告历史 + 板块分析
2. 事件详情（长描述，严重信号的下级子项）
3. 🟡异常偏离 → 交易所要求发公告
4. 利好出尽（👍事件 + 领跌）
5. 仅事件（利好/中性事件展示）
6. 弱信号

## 十二、事件系统详解

### 12.1 架构

```
check_events(code, name, days=30)
  → 东财直连API np-anotice-stock.eastmoney.com
  → 分类: zc(质押) / qy(减持/诉讼/立案/冻结/处罚/违规/亏损) / ba(增持/回购/中标/预增/扭亏)
  → 否定前缀过滤: "未/无/不"开头的关键词不标利空
  → 返回 (emoji, 纯emoji, 短标签, 长描述)

_make_event_tag(title, typ)
  → 关键词→短标签（≤6字）
  → zc: 解质/质押延期/质押
  → qy: 减持完毕/减持计划/诉讼/立案/冻结/预亏
  → ba: 中标/增持/回购/预增/扭亏
```

### 12.2 三版本输出

| 版本 | 用途 | 示例 |
|------|------|------|
| 纯emoji | （保留，暂未使用） | "🔒👎" |
| 短标签 | 表格事件列 | "🔒质押 \| 👎减持完毕" |
| 长描述 | 提示区事件详情 | "🔒控股股东解除质押 | 👎部分董事减持计划完成.." |

### 12.3 缓存

- `EVT_CACHE`：进程内缓存，key = `{代码}_{天数}`，避免重复请求
- `EVT_LONG_DETAILS`：长描述映射字典，供提示区查找

### 12.4 数据结构

```python
# check_events 返回
("⚡",           # emoji（历史兼容）
 "🔒👎",         # 纯emoji
 "🔒质押 | 👎减持完毕",  # 短标签
 "🔒控股股东解质 | 👎部分董事减持完成")  # 长描述

# 缓存存储
EVT_CACHE[key] = ("⚡", short_str, tag_str, long_str)
EVT_LONG_DETAILS[lbl] = long_str  # 提示区使用
```
